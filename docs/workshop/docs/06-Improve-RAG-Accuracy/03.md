
# 6.3 Update Copilot With Semantic Ranking

The next step is to update your API to use the semantic ranking capability. This involves updating how SOW chunks (blocks of content extracted from each SOW uploaded into the application) are found and retrieved, leveraging the semantic ranking operator directly in SQL.

## Review the function

Following the function calling pattern used by your copilot agent to retrieve data from the database, a Python function is used to execute semantic ranking queries. Within the `src/api/app/functions/chat_functions.py` file, the `find_sow_chunks_with_semantic_ranking` function is provided for executing queries that use the `azure_ai.rank()` semantic operator directly in SQL. Open it now in Visual Studio Code and explore the code. You can also expand the section below to see the code inline.

???+ info "Find SOW Chunks with Semantic Ranking code"

    ```python linenums="1" title="src/api/app/functions/chat_functions.py"
    async def find_sow_chunks_with_semantic_ranking(self, user_query: str, sow_id: Optional[int] = None, max_results: int = 3):
        """
        Retrieves content chunks similar to the user query for the specified SOW.
        """

        # Get the embeddings for the user query
        query_embeddings = await self.__create_query_embeddings(user_query)

        # Create a vector search query
        cte_query = f"SELECT array_agg(content) AS contents FROM ("
        cte_query += f" SELECT content FROM sow_chunks"
        cte_query += f" WHERE sow_id = {sow_id}" if sow_id is not None else ""
        cte_query += f" ORDER BY embedding <=> '{query_embeddings}'"
        cte_query += f" LIMIT 10"
        cte_query += f")"

        # Create the semantic ranker query
        query = f"""
        WITH vector_results AS (
            {cte_query}
        ),
        ranked AS (
            SELECT r.idx, r.relevance
            FROM vector_results, azure_ai.rank('{user_query}', contents, '{self.model}') AS r(idx, relevance)
        )
        SELECT contents[ranked.idx + 1] AS content, ranked.relevance
        FROM ranked, vector_results
        ORDER BY ranked.relevance DESC
        LIMIT {max_results};
        """

        rows = await self.__execute_query(f'{query};')
        return [jsonable_encoder(row) for row in rows]
    ```

1. **Generate embeddings**: Azure OpenAI generates embeddings representing the user query.
2. **Create vector search query**: The function creates a vector search query to select the `content` field from the `sow_chunks` table, ordering by semantic similarity.
3. **Semantic reranking**: The query uses a CTE to aggregate the top results, then calls the `azure_ai.rank()` operator directly in SQL to semantically rerank the results based on the user query. The most relevant results are returned, ordered by semantic relevance.
4. **Return the results**: The query results are extracted and returned to the LLM.

## Explore semantic ranker integration

To use the semantic ranking functionality instead of the standard vector search to retrieve SOW chunks, review how the function is assigned to your copilot agent's tools collection. In the `src/api/app/routers/completions.py` file, the `tools` array should include the `find_sow_chunks_with_semantic_ranking` function. This ensures that the copilot uses semantic ranking for more accurate and contextually relevant results.
