## Semantic Operators in Azure PostgreSQL

Azure Database for PostgreSQL now supports a set of semantic operators as part of the Azure AI extension. These operators enable advanced AI-powered search and ranking capabilities directly within your SQL queries. You can read more in the official documentation: [Azure AI semantic operators for PostgreSQL](https://learn.microsoft.com/en-us/azure/postgresql/flexible-server/generative-ai-azure-ai-semantic-operators).

### The `rank` Operator

The `azure_ai.rank()` operator is a powerful semantic operator that allows you to re-rank a set of results based on their semantic relevance to a given query. It takes a user query and an array of candidate texts, and returns a set of indices and relevance scores indicating how well each candidate matches the query.

**How it works:**

- You provide a query and an array of candidate texts (such as content chunks from your database).
- The operator uses a large language model (LLM) to evaluate the semantic similarity between the query and each candidate.
- It returns a set of results, each with an index (pointing to the candidate in the array) and a relevance score (higher means more relevant).

This operator is used in your SQL queries to ensure that the most contextually relevant results are surfaced for the user, going beyond simple keyword or vector similarity. It enables more accurate and meaningful search experiences in AI-powered applications.

For more details and advanced usage, see the [official Microsoft documentation](https://learn.microsoft.com/en-us/azure/postgresql/flexible-server/generative-ai-azure-ai-semantic-operators).

## How Semantic Ranking Works

Semantic ranking is now performed directly in your SQL queries using the `azure_ai.rank()` operator from the Azure AI extension for PostgreSQL. This operator allows you to call the semantic ranking model inline as part of your query logic, enabling you to combine vector search and semantic reranking in a single SQL statement.

In the backend, the `find_sow_chunks_with_semantic_ranking` function in `chat_functions.py` demonstrates this approach. The function generates vector embeddings for the user query, then constructs and executes a SQL query that:

- Performs a vector search to retrieve the most relevant content chunks from the `sow_chunks` table.
- Uses a Common Table Expression (CTE) to aggregate the top results into an array.
- Calls the `azure_ai.rank()` operator directly in the SQL to semantically rerank the results based on the user query.
- Returns the top reranked results, ordered by semantic relevance.

Here is a sample query that demonstrates how you can combine vector search and semantic reranking in a single SQL statement:

<!-- markdownlint-disable MD046 -->
```SQL
WITH vector_results AS (
    SELECT array_agg(content) AS contents FROM (
        SELECT content FROM sow_chunks
        ORDER BY embedding <=> azure_openai.create_embeddings('embeddings', 'cost management and optimization')::vector
        LIMIT 10
    )
),
ranked AS (
    SELECT r.idx, r.relevance
    FROM vector_results, azure_ai.rank('cost management and optimization', contents, '<model_name>') AS r(idx, relevance)
)
SELECT contents[ranked.idx + 1] AS content, ranked.relevance
FROM ranked, vector_results
ORDER BY ranked.relevance DESC
LIMIT 3;
```

This method improves the relevance of search results by leveraging semantic understanding, ensuring that the most contextually appropriate content is returned for user queries. You can review the logic in the `find_sow_chunks_with_semantic_ranking` function in your backend code to see how this is implemented in practice.
