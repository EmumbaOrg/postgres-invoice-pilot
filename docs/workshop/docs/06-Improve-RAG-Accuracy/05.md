
# 6.5 Graph Database Creation

Azure Database for PostgreSQL with the Apache AGE extension enables you to analyze complex relationships in your data using graph models. The graph database is created and loaded as part of the deployment process, integrating seamlessly with the Invoice Pilot copilot. This allows the copilot to query relationships between vendors, SOWs, and invoices using Cypher queries, in addition to standard SQL queries, improving retrieval-augmented generation (RAG) accuracy.

## Graph Structure

The graph models the relationships between:

- **Vendors** (nodes)
- **SOWs** (nodes)
- **Invoices** (edges between vendors and SOWs)

This structure allows you to efficiently query which vendors are associated with which SOWs and invoices, and to analyze invoice properties and payment status.

## How the Graph is Created

The graph is created and loaded by the SQL script `scripts/sql/load-graph-from-tables.sql`, which is executed automatically as part of the deployment process by the post-deployment azd hook (`azd-hooks/postdeploy.ps1`).

This script reads data from the `vendors`, `sows`, and `invoices` tables and creates:

- **Vendor nodes** for each vendor
- **SOW nodes** for each SOW
- **has_invoices edges** between vendors and SOWs, with invoice properties as edge attributes

It uses the Apache AGE extension's Cypher support to create the graph and populate it with nodes and edges.

???+ "Review the full SQL script for better understanding"
    ```sql
    -- scripts/sql/load-graph-from-tables.sql
    -- This script creates graph nodes and edges in Apache AGE from SQL tables: vendors, sows, has_invoices

    SET search_path = ag_catalog, "$user", public;

    -- 1. Create vendor vertices from vendors nodes
    SELECT * FROM ag_catalog.create_graph('vendor_graph');

    DO $function_body$
    DECLARE
        rec RECORD;
        cypher_body text;
        sql text;
    BEGIN
        FOR rec IN SELECT * FROM vendors LOOP
            cypher_body := format($create$
                CREATE (v:vendor {
                    id: %L,
                    name: %L,
                    address: %L,
                    website: %L,
                    type: %L,
                    contact_name: %L,
                    contact_email: %L,
                    contact_phone: %L
                })
            $create$,
                rec.id,
                rec.name,
                rec.address,
                rec.website,
                rec.type,
                rec.contact_name,
                rec.contact_email,
                rec.contact_phone
            );
            sql := 'SELECT * FROM cypher(''vendor_graph'', $AGE$'
                || cypher_body ||
                '$AGE$) AS (result agtype);';
            EXECUTE sql;
        END LOOP;
    END
    $function_body$;

    -- 2. Create sow vertices from sows table
    DO $function_body$
    DECLARE
        rec RECORD;
        cypher_body text;
        sql text;
    BEGIN
        FOR rec IN SELECT id, number, vendor_id, start_date, end_date, budget FROM sows LOOP
            cypher_body := format($create$
                CREATE (s:sow {
                    id: %L,
                    number: %L,
                    vendor_id: %L,
                    start_date: %L,
                    end_date: %L,
                    budget: %L
                })
            $create$,
                rec.id,
                rec.number,
                rec.vendor_id,
                rec.start_date,
                rec.end_date,
                rec.budget
            );
            sql := 'SELECT * FROM cypher(''vendor_graph'', $AGE$'
                || cypher_body ||
                '$AGE$) AS (result agtype);';
            EXECUTE sql;
        END LOOP;
    END
    $function_body$;

    -- 3. Create has_invoices edges from has_invoices table
    DO $function_body$
    DECLARE
        rec RECORD;
        cypher_body text;
        sql text;
    BEGIN
        FOR rec IN 
            SELECT 
                id, 
                vendor_id AS start_id, 
                'vendor' AS start_vertex_type, 
                sow_id AS end_id, 
                'sow' AS end_vertex_type, 
                number, 
                amount, 
                invoice_date, 
                payment_status 
            FROM invoices 
        LOOP
            cypher_body := format($create$
                MATCH (v:%I), (s:%I)
                WHERE v.id = %L AND s.id = %L
                CREATE (v)-[rel:has_invoices {
                    id: %L,
                    amount: %L,
                    number: %L,
                    invoice_date: %L,
                    payment_status: %L
                }]->(s)
            $create$,
                rec.start_vertex_type,
                rec.end_vertex_type,
                rec.start_id,
                rec.end_id,
                rec.id,
                rec.amount,
                rec.number,
                rec.invoice_date,
                rec.payment_status
            );
            sql := 'SELECT * FROM cypher(''vendor_graph'', $AGE$'
                || cypher_body ||
                '$AGE$) AS (result agtype);';
            EXECUTE sql;
        END LOOP;
    END
    $function_body$;
    ```

## Verifying the Graph

After deployment, you can verify the graph and explore relationships using Cypher queries in pgAdmin or any SQL client. For example, to view vendors and SOWs with invoice details:

    ```sql 
    SET search_path = ag_catalog, "$user", public;
    SELECT * FROM ag_catalog.cypher('vendor_graph', $$
    MATCH (v:vendor)-[rel:has_invoices]->(s:sow)
    RETURN v.id AS vendor_id, v.name AS vendor_name, s.id AS sow_id, s.number AS sow_number, rel.payment_status AS payment_status, rel.amount AS invoice_amount
    $$) as graph_query(vendor_id BIGINT, vendor_name TEXT, sow_id BIGINT, sow_number TEXT, payment_status TEXT, invoice_amount FLOAT);
    ```
