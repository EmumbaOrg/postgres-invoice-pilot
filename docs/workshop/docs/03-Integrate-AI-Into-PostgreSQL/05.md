# 3.5 Triggers to Generate Embeddings

Previously, we saw an overview of how an `embedding` column of `vector` type is added to some of our tables in the database. In this section, we’ll see the use of trigger functions that automatically generate embeddings for the `embedding` column using `azure_openai`. This ensures that embeddings are created automatically whenever new data is inserted into the tables.

The trigger for each table is created automatically as part of the `scripts/sql/deploy-database-tables.sql` file, which is executed during the initial setup process.

=== "deliverables"

    ```sql
    CREATE OR REPLACE FUNCTION deliverables_insert_trigger_fn()
    RETURNS trigger AS $$
    BEGIN
      IF NEW.description IS NOT NULL THEN
        NEW.embedding := azure_openai.create_embeddings('embeddings', NEW.description, throw_on_error => FALSE, max_attempts => 1000, retry_delay_ms => 2000);
      END IF;
      RETURN NEW;
    END;
    $$ LANGUAGE plpgsql;
    -- setup INSERT TRIGGER to call function
    CREATE TRIGGER deliverables_insert_trigger
    BEFORE INSERT ON deliverables
    FOR EACH ROW
    EXECUTE FUNCTION deliverables_insert_trigger_fn();
    ```

=== "invoice_validation_results"

    ```sql
    CREATE OR REPLACE FUNCTION invoice_validation_results_insert_trigger_fn()
    RETURNS trigger AS $$
    BEGIN
      IF NEW.result IS NOT NULL THEN
        -- Create embeddings for the result field
        NEW.embedding := azure_openai.create_embeddings('embeddings', NEW.result, throw_on_error => FALSE, max_attempts => 5, retry_delay_ms => 2000);
        
        -- Generate summary using Azure Cognitive Services and store in result_summary field
        NEW.result_summary := azure_cognitive.summarize_abstractive(NEW.result, 'en', 2);
      END IF;
      RETURN NEW;
    END;
    $$ LANGUAGE plpgsql;
    -- setup INSERT TRIGGER to call function
    CREATE TRIGGER invoice_validation_results_insert_trigger
    BEFORE INSERT ON invoice_validation_results
    FOR EACH ROW
    EXECUTE FUNCTION invoice_validation_results_insert_trigger_fn();
    ```

With these triggers in place, the database is now equipped to automatically generate vector embeddings every time new content is added to these tables. This ensures consistency, eliminates the need for manual embedding generation, and prepares your system for seamless real-time integration with applications powered by `azure_openai`.
